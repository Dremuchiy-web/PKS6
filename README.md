## Описание проекта

MAD Shop - это мобильное приложение для интернет-магазина, сверстанное на Flutter по макету из Figma. Проект демонстрирует процесс переноса готового дизайна в код с соблюдением принципов адаптивной верстки и UI/UX лучших практик.

## Структура проекта

```
lib/
  ├── main.dart                 # Точка входа приложения
  ├── screens/                  # Экраны приложения
  │   ├── splash_screen.dart    # Экран загрузки
  │   ├── onboarding/           # Экраны авторизации
  │   │   ├── create_account_screen.dart
  │   │   ├── login_screen.dart
  │   │   └── password_screen.dart
  │   ├── shop_screen.dart      # Главный экран с каталогом товаров
  │   ├── product_screen.dart   # Экран детальной информации о товаре
  │   ├── favourites_screen.dart # Экран избранного
  │   └── cart_screen.dart      # Экран корзины
  ├── widgets/                  # Переиспользуемые компоненты
  │   └── product_card.dart     # Карточка товара
  ├── services/                 # Сервисы приложения
  │   ├── cart_service.dart     # Сервис управления корзиной
  │   └── favorites_service.dart # Сервис управления избранным
  └── theme/                    # Стили и темы
      ├── colors.dart           # Цветовая палитра
      └── text_styles.dart      # Стили текста
```

## Основные функции

- ✅ **Поиск товаров** - поиск по названию и описанию
- ✅ **Избранное** - добавление/удаление товаров в избранное
- ✅ **Корзина** - добавление товаров с выбором количества, цвета и размера
- ✅ **Навигация** - переходы между экранами
- ✅ **Выход из аккаунта** - кнопка выхода с подтверждением
- ✅ **Авторизация** - регистрация и вход в аккаунт

## Сверстанные экраны

### 1. Splash Screen
- **Описание:** Экран загрузки с логотипом приложения
- **Элементы:** Белый круг с синей иконкой сумки
- **Навигация:** Автоматический переход на экран регистрации

### 2. Onboarding экраны
- **Create Account:** Регистрация с полями Email, Password, Phone
- **Login:** Вход в аккаунт с приветствием
- **Password:** Экран ввода пароля
- **Навигация:** Переходы между экранами авторизации

### 3. Главный экран (Shop Screen)
- **Описание:** Отображает каталог товаров в виде сетки (GridView)
- **Элементы:**
  - AppBar с названием приложения и иконкой корзины
  - Заголовок секции с количеством товаров
  - Сетка карточек товаров (2 колонки)
  - Каждая карточка содержит изображение, название и цену
- **Навигация:** При нажатии на карточку товара происходит переход на экран детальной информации

### 4. Экран товара (Product Screen)
- **Описание:** Детальная информация о выбранном товаре
- **Элементы:**
  - Большое изображение товара
  - Название товара
  - Цена
  - Описание товара
  - Кнопка "Добавить в корзину"
- **Функциональность:** 
  - Добавление товара в корзину
  - Отображение уведомления об успешном добавлении
  - Изменение состояния кнопки после добавления

### 5. Экран избранного (Favourites Screen)
- **Описание:** Отображает только избранные товары
- **Элементы:**
  - Сетка товаров с красными сердечками
  - Пустое состояние, если нет избранных товаров
- **Функциональность:** 
  - Автоматическое обновление при изменении избранного
  - Удаление из избранного

### 6. Экран корзины (Cart Screen)
- **Описание:** Отображение выбранных товаров и оформление заказа
- **Элементы:**
  - Список товаров в корзине
  - Для каждого товара: изображение, название, цена, количество
  - Кнопки управления количеством (увеличить/уменьшить)
  - Кнопка удаления товара
  - Итоговая сумма заказа
  - Кнопка "Оформить заказ"
- **Состояния:**
  - Пустая корзина с сообщением
  - Корзина с товарами

## Сервисы

### CartService
- Управление корзиной покупок
- Добавление/удаление товаров
- Изменение количества
- Подсчет общей стоимости
- Синхронизация между экранами

### FavoritesService
- Управление избранными товарами
- Добавление/удаление из избранного
- Синхронизация между экранами

## Использованные элементы и виджеты

### Компоновка:
- `Container` - для создания контейнеров с отступами и стилями
- `Padding` - для управления отступами
- `Column` - для вертикального размещения элементов
- `Row` - для горизонтального размещения элементов
- `GridView` - для отображения сетки товаров
- `ListView` - для отображения списка товаров в корзине
- `Stack` - для наложения элементов (например, бейдж на иконке корзины)
- `SingleChildScrollView` - для прокрутки контента

### Стилизация:
- `Card` - для карточек товаров
- `ElevatedButton` - для кнопок действий
- `IconButton` - для иконок-кнопок
- `Divider` - для разделителей
- `SnackBar` - для уведомлений

### Навигация:
- `Navigator.push()` - для перехода между экранами
- `MaterialPageRoute` - для создания маршрутов

### Тема и стили:
- `AppColors` - централизованное управление цветами
- `AppTextStyles` - централизованное управление стилями текста
- `ThemeData` - единая тема приложения

## Цветовая палитра

- **Primary:** `#2196F3` (синий) - основной цвет приложения
- **Accent:** `#FFC107` (желтый) - акцентный цвет
- **Background:** `#FFFFFF` (белый) - фон приложения
- **Surface:** `#FFFFFF` (белый) - фон карточек
- **Text Primary:** `#000000` (черный) - основной текст
- **Text Secondary:** `#757575` (серый) - вторичный текст
- **Favorite:** `#E91E63` (розовый) - цвет избранного

## Навигация между экранами

1. **Главный экран → Экран товара:**
   - Нажатие на карточку товара
   - Используется `Navigator.push()` с `MaterialPageRoute`

2. **Главный экран → Корзина:**
   - Нажатие на иконку корзины в AppBar
   - Используется `Navigator.push()` с `MaterialPageRoute`

3. **Возврат назад:**
   - Используется стандартная кнопка "Назад" в AppBar
   - Или жест свайпа назад (на поддерживаемых устройствах)

## Отличия от макета Figma

Поскольку ссылка на макет Figma не была предоставлена, приложение было создано на основе общих принципов дизайна интернет-магазинов и требований задания. Основные отличия, которые могут возникнуть при сравнении с оригинальным макетом:

1. **Изображения товаров:** Используются placeholder изображения вместо реальных изображений из Figma
2. **Шрифты:** Используются стандартные шрифты Flutter вместо кастомных шрифтов из макета
3. **Иконки:** Используются Material Icons вместо кастомных иконок
4. **Анимации:** Базовые переходы без сложных анимаций, которые могут быть в макете

## Анализ UI/UX решений

### Удачные решения:

1. **Централизованное управление стилями:**
   - Все цвета и стили текста вынесены в отдельные файлы
   - Легко изменять дизайн всего приложения из одного места
   - Соответствует принципам чистого кода

2. **Адаптивная сетка товаров:**
   - Использование GridView обеспечивает корректное отображение на разных размерах экранов
   - 2 колонки оптимальны для мобильных устройств

3. **Интуитивная навигация:**
   - Понятные переходы между экранами
   - Стандартные элементы Material Design знакомы пользователям

4. **Обратная связь:**
   - SnackBar уведомления информируют пользователя о действиях
   - Изменение состояния кнопок показывает текущий статус

5. **Пустые состояния:**
   - Корзина показывает понятное сообщение, когда пуста
   - Улучшает пользовательский опыт

### Что можно улучшить:

1. **Загрузка изображений:**
   - Добавить кэширование изображений
   - Использовать пакет `cached_network_image` для оптимизации

2. **Анимации:**
   - Добавить плавные переходы между экранами
   - Анимации при добавлении товаров в корзину

3. **Поиск и фильтры:**
   - ✅ Поиск по товарам реализован
   - Фильтрация по категориям и цене (можно добавить)

4. **Состояние приложения:**
   - ✅ Используются сервисы (CartService, FavoritesService) для управления состоянием
   - ✅ Глобальное состояние корзины и избранного синхронизируется между экранами

5. **Обработка ошибок:**
   - Улучшить обработку ошибок загрузки изображений
   - Добавить индикаторы загрузки

6. **Адаптивность:**
   - Оптимизация для планшетов (больше колонок в сетке)
